# hai7 人名括注与年号检查：实施方案

## 一、数据实际结构（`data/hai7.json`）

经抽样与统计，当前文件包含：

| 键 | 说明 | 规模（约） | 备注 |
|----|------|------------|------|
| `person_birth_death` | 人名 → 记录数组 | 约 10027 个人名 | 每行含 `raw`（生卒年），可选 `label`（外文名等） |
| `era_years` | 年号 → 记录数组 | 约 529 个年号 | 每行含 `raw`（公历起止年），可选 `label`（朝代/说明） |

- **多条记录**：部分人名（约 40+）与年号（约 100+）下有多条数据（如「亨利四世」「上元」），需在结果中单独标记「多义项」，与「与表不符」可叠加。
- **raw 形态**：除常见 `1573—1620` 外，还有 `910—？`、`784—？`、`前134—前129`（公元前）、以及含「或」的写法（如 `673或683—727`）。核对时对 **raw 字符串** 做空白与连接号归一后比较；复杂写法须与表中 **完全一致**（归一后）才算匹配。
- **label**：人名约 3700+ 条带 `label`；括注中若写出外文名，须与对应行的 `label` 一致（归一后）。

以上结论用于校准规则：**不能**假设 raw 均为「四位整数—四位整数」；年号推算已支持「前×××」形式的公元前年份。

## 二、在扩展中的集成方式

1. **数据加载**：在 `tableLoader` 中懒加载 `data/hai7.json`，与 `getHai7Data()` 暴露给扫描器。
2. **自定义检查预置**：在「替换表」侧栏增加两项预置（与其它预置一样可勾选、排序）：
   - **hai7 人名括注**（`preset_hai7_person`）
   - **hai7 年号与纪年**（`preset_hai7_era`）
3. **扫描实现**：`documentScannerHai7.ts`（不走通用「词典键→值」扫描，与 `preset_gjtg` 类似）。
4. **命令入口**：「检查字词 → 自定义检查」中，对已勾选预置逐条调用对应扫描函数，结果合并入字词检查树。

## 三、人名括注检查（实现要点）

1. **匹配形式**：表中每个人名作为字面串（按长度降序组成 alternation，避免短名抢长名），后接半角或全角括号，括注内为非贪婪匹配至成对闭括号：`Name（…）` / `Name(…)`。
2. **括注解析**：
   - 若含 **一个** 英文/中文逗号分隔段：视为 **仅 raw**（生卒年）。
   - 若含 **两个及以上** 段（以首逗号分割）：**严格** 第一段为 `label`，其后合并为 `raw`（允许 raw 内含逗号）。
3. **与表核对**：在 `person_birth_death[name]` 中查找是否存在一行，使 `raw` 一致且（若括注含 label）`label` 一致。
4. **标记**：
   - **与表不符**：无一行匹配。
   - **表内多义项**：该人名在表中记录数 > 1（即使本次括注能匹配其中一行，仍提示需人工核对）。
5. **归一化**：括号、逗号全半角；连接号 `—－-–` 统一；`label` 使用 NFKC 并压缩空白，减轻全角半角与兼容字符差异。

## 四、年号检查（实现要点）

1. **括注类**（`年号` + 可选 `年` + 括注）：与人名类似，解析 `label` / `raw` 顺序及与 `era_years[年号]` 的匹配；多行 raw 时叠加「多义项」标记。
2. **汉字纪年 + 无括注**：`年号` + `[元一二…百千]` + `年`。对表中 **每一行** raw 取左端点作为「元年」公历年份，计算「第 n 年」= `元年 + (n−1)`，再判断是否落在 **该行** raw 区间内；若 **没有任何一行** 能容纳该年 → **超出 raw 范围**。若 **多行** 同时容纳 → **多义项**（与超范围提示可组合）。
3. **汉字纪年 + 括注阿拉伯数字**：在（2）的推算下，括注可为单一年份或 `a—b` 区间（阿拉伯数字与连接号归一），须 **唯一** 对应到合法推算结果之一；否则标为 **不一致**。
4. **仅「年号+年」**（如「万历年」「万历年」出现在「万历年间」前三字）：不标为错误，仅 **信息提示**，提醒结合上下文核对；若同一位置已被其它规则处理（如已含括注），则不重复报。

**扫描顺序**：先处理「汉字年+括注」整段，再「汉字年无括注」，再「仅括注」，最后「年号+年」提示，并用区间占用避免同一位置重复多条冲突结果。

## 五、性能与限制

- 人名、年号键名联合成正则 alternation（万级/千级），在一般章节长度下可接受；极大文件若变慢，可后续改为分块正则或 Aho-Corasick。
- 括注内嵌套括号、表格跨行等极端排版可能漏检或误截断，以编辑常见「单行括注」为准。

## 六、维护建议

- 更新书稿用表时，直接替换或合并 `data/hai7.json`，保持键为正文使用的标准写法人名/年号。
- 新增年号时注意：同一汉字年号多时期须在表中分多行，以便多义项提示生效。
